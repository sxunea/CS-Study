# 5.1.1 시간 복잡도

시간 복잡도란 문제를 해결하는 데 걸리는 시간과 입력의 함수 관계를 나타낸다. 어떠한 알고리즘의 로직이 얼마나 오랜 시간이 걸리는지를 나타내는 데 쓰이며, 보통 **빅오 표기법**으로 나타낸다. 

Big-O(빅-오)표기법은 점근으로 최악의 경우, Big-Ω(빅-오메가)는 하한 점근으로 최선의 경우, Big-θ(빅-세타)표기법은 이 둘의 평균 정도를 나타낸다. 

## Big-O
빅오 표기법이란 입력 범위 n을 기준으로 해서, 로직이 몇 번 반복되는지 나타내는 것이며, 다음과 같은 특징을 가진다. 
- 상수항을 무시
    - 어떤 알고리즘이 O(N+5)의 복잡도를 가졌으면 상수를 생략해 O(N)으로 표기한다.

- 계수도 무시
    - 어떤 알고리즘이 O(3N)의 복잡도를 가졌으면 계수를 생략해 O(N)으로 표기한다.

- 최고차항만 표기
    - 어떤 알고리즘이 O(3N^3+2N^2+N+5)의 복잡도를 가졌으면 O(N^3)으로 표기한다.


## Big-O 표기법으로 나타내는 시간 복잡도의 종류
![big o](https://github.com/sxunea/CS-Study/assets/81434152/914fc670-1841-4740-b08b-c7c0643949b7)

- **O(1)** : constant complexity - 입력 값이 늘어나도 소요 시간이 늘어나지 않는다 
- **O(log n)** : logarithmic complexity - 대표적으로 BST 탐색 알고리즘
- **O(n)** : linear complexity - 입력 값이 늘어나면 소요 시간이 그에 따라 일정하게 늘어난다
- **O(nlog n)**
- **O(n^2)** : quadratic complexity 
- **O(2n)** : exponential complexity - 대표적으로 피보나치 수열
