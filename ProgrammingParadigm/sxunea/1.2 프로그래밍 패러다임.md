
# 1.2 프로그래밍 패러다임
프로그래밍 패러다임이란 프로그래머에게 프로그래밍의 관점을 갖게 해주는 역할을 하는 **개발 방법론**이다. 예를 들어 객체 지향 프로그래밍은 개발자들이 프로그램을 상호 작용하는 객체들의 집합으로 볼 수 있게 하는 반면에, 함수형 프로그래밍은 상태 값을 지니지 않는 함수 값들의 연속을 생각할 수 있게 해준다. 

## 선언형 프로그래밍과 명령형 프로그래밍

프로그래밍 패러다임은 크게 선언형, 명령형으로 나눈다. **명령형 프로그래밍은 어떻게 할 지**를 표현하고, **선언형 프로그래밍은 무엇을 할 건지** 표현한다. 다시 선언형은 함수형이라는 하위 집합을 갖는다. 또, 명령형은 다시 객체 지향, 절차 지향으로 나눈다. 따라서 프로그래밍 패러다임에 대해 잘 알아두면 프로그램, 프로그래밍 언어를 구성하는 방식을 이해할 수 있는 관점을 갖게 해주므로 좋은 프로그램을 만드는 데에도 도움을 준다.
