
# 1.2.2 객체지향 프로그래밍

**객체 지향 프로그래밍**(Object-Oriented Programming)은 객체들의 집합으로 프로그램의 상호작용을 표현하며 데이터를 객체로 취급하여 객체 내부의 선언된 메소드를 활용하는 방식을 말한다. 

## 객체란
객체(Object)란 물리적으로 존재하거나 추상적으로 생각할 수 있는 것 중에서 자신의 속성을 가지고 있고 다른것과 식별 가능한 것을 말한다. 객체들의 공통적인 특성을 파악한 후, 하나의 묶음으로 만든다. 각각의 객체를 추상화시켜 속성과 기능으로 분류한 후에 이것을 다시 각각 변수와 함수로 정의한다.
## 객체 지향 프로그래밍의 특징
### 추상화
추상화(abstraction)란 복잡한 시스템으로부터 핵심적인 개념 또는 기능을 간추려내는 것을 의미한다. 객체들의 공통적인 특성을 파악한 후, 하나의 묶음으로 만든다.

 - 인터페이스 
 - 추상 클래스

### 캡슐화
캡슐화(encapsulation)는 객체의 속성과 메서드를 하나로 묶고 일부를 외부에 감추어 은닉하는 것을 말한다. 외부에서 객체를 손상시키는 일을 방지할 수 있도록, 객체의 세부적인 동작에 대한 구현을 감춘다. 
 - 접근 제한자 활용

### 상속성
상속성(inheritance)은 상위 클래스의 특성을 하위 클래스를 이어받아서 재사용하거나 추가, 확정하는 것을 말한다. 자식 클래스를 외부로부터 은닉하는 캡슐화의 개념을 보유한다.


### 다형성
다형성(polymorphism)은 하나의 메서드나 클래스가 다양한 방법으로 동작하는 것을 말한다. 같은 명령을 각자 연결된 객체에 의존해서 해석하는 걸 뜻한다.
 - overloading : 같은 이름을 가진 메서드를 여러 개 두고 상황에 따라 다른 역할을 수행시키는 것을 말한다.
 - overriding : 상위 클래스로부터 상속받은 메서드를 하위 클래스가 재정의하는 것을 말한다.

## 객체 지향 설계 원칙(SOLID)
- **S**RP : 단일 책임 원칙
- **O**CP : 개방 폐쇄 원칙
- **L**SP : 리스코프 치환 원칙
- **I**SP : 인터페이스 분리 원칙
- **D**IP : 의존관계 역전 원칙


## 객체지향 프로그래밍의 장단점
- 장점
    - 코드 재사용이 용이하다(반복되는코드 최소화)
    - 생산성이 높아진다
    - 유지보수에 용이하다

- 단점
    - 객체단위로 프로그램을 만들다보면, 프로그램의 용량이 증가할 수 있다
    - 순수함수의 구현을 위해 프로그래밍 하다보면 가독성이 떨어질 수 있다
