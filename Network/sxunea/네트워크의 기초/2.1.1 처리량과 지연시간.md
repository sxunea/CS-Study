# 2.1.1 처리량과 지연시간

**네트워크**란 **노드**(서버,라우터,스위치 등의 네트워크 장치)와 **링크**(유선 무선)가 서로 연결되어 있으며 리소스를 공유하는 집합을 말한다

## 처리량
**처리량**은 링크 내에서 성공적으로 전달된 데이터의 양을 말하며, **보통 얼만큼의 트래픽을 처리**했는지를 나타낸다
- 단위로는 **bps**, 초당 전송 또는 수신되는 비트 수를 활용한다
- 트래픽, 네트워크 장치간의 대역폭, 에러, 하드웨어 스펙등에 영향을 받는다

여기서 **트래픽**이란 특정 시점에 링크 내에 흐르는 데이터의 양을 말하는데, 트래픽이 많아진다는 건 흐르는 데이터가 많아졌다는 것, 따라서 처리량이 많아졌다는 것은 처리되는 트래픽이 많아졌다는 것이다

## 지연시간
**지연 시간**이란 **요청이 처리되는 시간**을 말하며 어떤 메시지가 두 장치 사이를 왕복하는 데 걸린 시간을 말한다 
- 매체 타입(유선 무선), 패킷 크기, 라우터의 패킷 처리 시간에 영향을 받는다

## 처리량과 지연 시간의 이용

지연 시간에 따라 사용자가 네트워크에서 데이터를 보내거나 받을 때 지연을 경험하게 된다. 또, 처리량에 따라 네트워크에 동시에 액세스할 수 있는 사용자 수가 결정된다

처리량이 적고 지연 시간이 긴 네트워크는 많은 양의 데이터를 전송 및 처리하는 데 어려움을 겪으며 이로 인해 정체가 발생하고 애플리케이션 성능이 저하된다. 반대로 처리량이 많고 지연 시간이 짧은 네트워크는 반응성이 뛰어나고 효율적이다. 

## 처리량과 지연 시간의 개선

지연 시간을 개선하려면 소스와 대상 간의 전파를 줄이고 전체 네트워크 대역폭을 늘려 처리량을 개선할 수 있다

- **캐싱**

   네트워킹에서의 캐싱은 자주 액세스하는 데이터를 사용자와 지리적으로 가까운 곳에 저장하는 프로세스를 말한다. 예를 들어 프록시 서버나 콘텐츠 전송 네트워크(CDN)에 데이터를 저장할 수 있다. 네트워크는 원래 소스에서 데이터를 검색해야 하는 경우보다 훨씬 빠르게 캐시된 위치에서 데이터를 전송, 수신, 검색할 수 있어 그 속도가 향상된다.


- **전송 프로토콜**

  특정 애플리케이션에 사용하는 전송 프로토콜을 최적화하여 네트워크 성능을 개선할 수 있다. TCP는 연결을 설정하고 오류 없이 데이터를 수신하는지 확인하는데 패킷 손실을 줄이는 것이 목표이기 때문에 TCP는 지연 시간이 더 길고 처리량이 더 많다. UDP는 패킷 손실이나 오류를 검사하지 않고 대신 여러 개의 중복 패킷을 전송하므로 지연 시간은 최소화되지만 처리량은 더 많다.


- **서비스 품질**

  서비스 품질(QoS) 전략을 사용하여 네트워크 성능을 관리하고 최적화할 수 있다
